---
title: Column Sizing & Density
description: Fit columns to their content with smart width, and control row/cell spacing with density presets.
---

Two React `<Table>` props tune layout: `autoColumnWidth` sizes columns to their
content, and `density` sets a spacing preset. Both are opt-in and leave existing
layouts unchanged when unset.

## Smart Column Width

`autoColumnWidth` measures each column's content and resolves final widths under
a chosen overflow policy. It is **off by default** — pass `true` for the default
policy, or an object to tune it.

```tsx
// Default policy: fit to container, leave underflow, sample 100 rows.
<Table table={table} autoColumnWidth />

// Tuned:
<Table
  table={table}
  autoColumnWidth={{
    sampleSize: 100,          // rows sampled for measurement (default 100)
    overflow: 'fit',          // 'fit' | 'scroll' (default 'fit')
    underflow: 'leave',       // 'leave' | 'distribute' | 'stretch' (default 'leave')
  }}
/>
```

### Overflow modes

When the columns' natural (content) widths sum to **more** than the container:

- **`fit`** — never scroll horizontally. Over-wide columns are squished and their
  cells **wrap**. Width is taken first from the columns with the most slack
  (largest content), proportional to each column's slack, and never below a
  floor (its `minSize`, or 48px).
- **`scroll`** — keep natural widths and allow the grid to scroll horizontally.

### Underflow modes

When natural widths sum to **less** than the container:

- **`leave`** — columns keep their natural width (the grid may not fill the container).
- **`distribute`** — extra space is spread proportionally across auto-sized columns
  in a **waterfall** that respects each column's `maxSize` as a **hard cap**. When
  a column would exceed its `maxSize` it is pinned at the cap and its leftover
  share cascades to the remaining uncapped columns, repeating until the space is
  spent or every column is capped. A gutter is left **only** when every auto column
  has hit its `maxSize`.
- **`stretch`** — the same waterfall, but `maxSize` is a **soft cap**. If space is
  still left after every column caps, all auto columns grow past their `maxSize`
  proportionally so the container fills **exactly** — never a gutter. Use this on
  modal/settings tables with a few capped columns where a trailing dead gutter
  looks broken.

Both `distribute` and `stretch` are total-preserving: final widths sum to the
container width (with the ±1px rounding remainder folded into the last grown
column), except `distribute` when every column is capped, which intentionally
leaves the remainder as a gutter.

### Per-column opt-out

A column keeps its width and is excluded from measurement and squishing when it
has an explicit `size`, or `enableAutoSize: false`:

```tsx
columnHelper.accessor('status', {
  header: 'Status',
  size: 120,               // explicit size opts out
})
columnHelper.accessor('actions', {
  header: '',
  enableAutoSize: false,   // keeps its width, never measured or squished
})
```

Computed widths flow through the table's `columnSizing` state, so pinned-column
offsets, the `<colgroup>`, and virtualization totals all stay in agreement. Once
you drag a column's resize handle, that column is excluded from further
auto-sizing so it never fights your manual width.

### Measuring rendered content

By default a column is measured by its raw accessor value (`String(row.getValue(id))`).
When the cell displays something different — a formatter or a custom `cell`
renderer — the raw value under-measures and the rendered text can clip (accessor
`19` rendered as `"$19.00 (Gold)"`). Two per-column overrides fix this:

```tsx
columnHelper.accessor('price', {
  header: 'Price',
  cell: ({ getValue }) => formatCurrency(getValue()),
  // Measure the string the cell actually renders:
  autoSizeText: (row) => formatCurrency(row.original.price),
})

columnHelper.accessor('trend', {
  header: 'Trend',
  cell: ({ getValue }) => <Sparkline value={getValue()} width={80} />,
  // Or provide an EXACT natural pixel width — bypasses text measurement
  // entirely (no padding or sort-indicator is added):
  autoSizeWidth: () => 80,
})
```

Precedence per column: `autoSizeWidth` (exact px, used verbatim) → `autoSizeText`
(measured string) → the raw accessor value. Both take a row and are sampled the
same way as normal values; `autoSizeText` still gets cell padding and the
sort-indicator allowance added, while `autoSizeWidth` does not.

### Async values (re-measure)

When a column's cell values arrive **after** mount — for example the rows render
first with placeholders (`$0`, `'-'`) and the real values are merged in from a
separate query — the first measurement sizes on the placeholder. Auto-sizing
corrects this automatically: it re-measures whenever the **row data reference
changes identity**. An async value merge produces a new `data` array, which
triggers a debounced (~60ms, to coalesce rapid updates) re-measure. Sorting,
filtering, and pagination do **not** re-measure — those change the derived row
model, not the `data` reference the hook watches.

Re-measure respects the provenance rule: only widths auto-sizing itself wrote are
updated; user-resized and persisted widths are never touched.

If you merge values **without** replacing the `data` array (mutating rows in
place, or any case where identity does not change), call the explicit escape
hatch once the merge is done:

```tsx
// After your async values are merged into the rows:
table.remeasureColumns()            // re-measures immediately (no debounce)
table.remeasureColumns('prices-loaded') // optional reason, forwarded on the event
```

`table.remeasureColumns(reason?)` emits a `'columns:remeasure'` event that any
active `autoColumnWidth` sizing listens for. It is a no-op when smart width is
off, and — like the automatic path — only re-sizes auto-owned columns.

### `minSize` is a hard floor

A column can **never** render below its `minSize` — core `getSize` clamps every
width (auto-computed, resized, or persisted) up to `minSize`. Auto-sizing honors
this: a column whose content is narrower than its `minSize` renders at `minSize`,
not tighter. If you set `minSize` purely as a flex floor and want auto-sizing to
size closer to the content, lower or omit it.

### Persisted widths

The hook **never overwrites a width it did not itself write this session** — a
user-dragged width or a width restored from persisted `columnSizing` is left
alone (it is treated as user-set, exactly like an in-session resize).

<Callout type="warn">
  **Persisting auto-sized widths neuters smart width.** If you save the whole
  `columnSizing` state (which includes the widths auto-sizing computed) and
  restore it into `columnSizing` on the next load, those restored widths look
  **user-set** to the provenance rule and are skipped — the column is no longer
  auto-measured, and async re-measure and `stretch` won't touch it. Prefer either
  **not** persisting `columnSizing` under `autoColumnWidth` (auto recomputes
  deterministically from content on every load), or persisting **only** the
  columns the user actually resized (mark them at resize time) and restoring just
  those, so genuinely auto columns stay auto.
</Callout>

### Limitations

- **Custom cell renderers.** Measurement uses each cell's raw string value. For
  columns rendering non-text content (badges, progress bars, custom components)
  or formatted text, provide [`autoSizeText` or `autoSizeWidth`](#measuring-rendered-content),
  or give the column an explicit `size` / `enableAutoSize: false`.
- **Row virtualization.** Wrapped rows have variable heights that the fixed-height
  virtualizer does not measure. When the table is row-virtualized, `overflow: 'fit'`
  falls back to `scroll` behavior (natural widths, horizontal scroll) instead of
  squishing and wrapping, and emits a one-time dev-only console warning. The
  supported way to get fit + wrap under virtualization is Pretext-measured row
  heights; otherwise disable virtualization or set `overflow: 'scroll'` to
  silence the warning.

## Density Presets

`density` maps to a token set (cell/header padding, row height, font size)
defined in the themes package. It is independent of the legacy `compact` prop,
which continues to work unchanged.

```tsx
<Table table={table} density="condensed" />
<Table table={table} density="regular" />
<Table table={table} density="spacious" />
```

| Preset       | Cell padding | Row height | Font size |
| ------------ | ------------ | ---------- | --------- |
| `condensed`  | 8 × 4px      | 32px       | 12px      |
| `regular`    | 14 × 10px    | 40px       | 13px      |
| `spacious`   | 20 × 14px    | 48px       | 14px      |

Each preset declares its full token set explicitly — including `regular` — so a
grid nested inside a differently scaled ancestor still renders its own density
rather than inheriting the ancestor's overridden variables.
