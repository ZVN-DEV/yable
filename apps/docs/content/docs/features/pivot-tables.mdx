Transform flat data into cross-tabulated views. Pivoting ships in Yable's MIT-licensed core as a programmatic row-model accessor.

<Callout type="info">
  Pivot ships as a programmatic row-model accessor (`table.getPivotRowModel()`).
  Rendering a pivot directly through `<Table>` is on the roadmap.
</Callout>

### How to Use

Declare a `pivotConfig` (row fields, optional column fields, and value aggregations), then read the aggregated rows off the model returned by `table.getPivotRowModel()`:

```typescript
const table = useTable({
  data,
  columns,
  pivotConfig: {
    rowFields: [{ field: 'region' }, { field: 'category' }],
    columnFields: [],
    valueFields: [{ field: 'sales', aggregation: 'sum' }],
  },
})

// Returns a real aggregated RowModel — read the rows directly.
const pivot = table.getPivotRowModel()

for (const row of pivot.rows) {
  // Aggregated values and pivot metadata live on `row.original`.
  console.log(row.original)
}
```

### Notes

- `table.getPivotRowModel()` returns a `RowModel` (`rows`, `flatRows`, `rowsById`); each row carries its aggregates and pivot metadata (`_pivotLabel`, `_pivotPath`, `_pivotDepth`, …) on `row.original`
- A full `PivotEngine` with row fields, column fields, value fields, subtotals, and grand totals is exported from `@zvndev/yable-core` (`PivotEngine`, `getPivotRowModel`, `generatePivotColumnDefs`, `getInitialPivotState`, `PivotConfig`)
- Value aggregations reuse the built-in [aggregation functions](/docs/features/aggregation) (`sum`, `mean`, `count`, …)
- Wiring the generated pivot columns into `<Table>` for turnkey rendering is on the roadmap
