# Code of Conduct

Yable is a small open-source project. We want contributors and users of all backgrounds and experience levels to feel welcome when they file issues, submit pull requests, or ask questions.

## Our standards

Contributors and maintainers agree to:

- Be respectful and considerate in discussions, reviews, and issue threads
- Assume good faith when reading someone else's words
- Give feedback on code, not on the person
- Help newer contributors when you have the bandwidth
- Keep conversations on-topic and focused on the project

We will not tolerate:

- Personal attacks, insults, or demeaning language
- Harassment in any form, public or private
- Posting others' private information without consent
- Sustained disruption of discussions or reviews
- Behavior that a reasonable person would find intimidating or hostile

## Scope

This applies to all project spaces: the GitHub repository, issues, pull requests, discussions, and any communication channels where someone is participating on behalf of the project.

## Reporting

If you see behavior that violates these standards, please report it privately. Use the email address listed on the GitHub organization profile, or open a GitHub Security Advisory if the issue is sensitive.

Reports will be handled confidentially. Maintainers will respond within a reasonable time, investigate the situation, and take whatever action they believe is appropriate. Actions can range from a private conversation to removing a contributor from the project.

## Enforcement

Maintainers are responsible for clarifying and enforcing these standards. They may edit or remove comments, commits, pull requests, and issues that do not follow this code of conduct, and may ban contributors whose behavior is harmful to the community.

Maintainers who violate this code of conduct are held to the same standards as anyone else.

## Questions

If something is unclear, open a GitHub discussion or email the maintainers. We would rather clarify than argue.
